## 测试用例

### 测试 1: 基础映射继承

**目的**: 验证耗材 5-8 正确从映射的挤出机继承参数

**步骤**:
1. 取消所有耗材的参数覆盖（retraction_length, z_hop, z_hop_types, wipe_distance）
2. 切片
3. 使用 analyze_gcode_params.py 分析 G-code

**预期结果**:

| 耗材 | 映射到 | retraction_length | z_hop | z_hop_types |
|------|--------|-------------------|------|------------|
| F5 | E1 | 0.8 | 0.2 | Normal |
| F6 | E2 | 1.0 | 0.3 | Spiral |
| F7 | E3 | 1.2 | 0.4 | Normal |
| F8 | E4 | 1.4 | 0.5 | Spiral |

**通过**: [ ] 是 [ ] 否

**日志验证**:
```
DEBUG_MAP_INDICES_RESULT: retraction_length [0->0 1->1 2->2 3->3 4->0 5->1 6->2 7->3 ]
```

---

### 测试 2: 耗材覆盖优先级

**目的**: 验证勾选覆盖的耗材使用自己的值，**不**继承挤出机值

**步骤**:
1. 为耗材 5 启用 retraction_length 覆盖，   设置值为 2.0
2. 保持耗材 6-8 不勾选覆盖
3. 切片
4. 验证 G-code

**预期结果**:
- F5 的 retraction_length = 2.0（使用覆盖值）
- F6 的 retraction_length = 1.0（继承 E2）
- F7 的 retraction_length = 1.2（继承 E3）
- F8 的 retraction_length = 1.4（继承 E4）

**通过**: [ ] 是 [ ] 否

---

### 测试 3: 修改挤出机参数

**目的**: 验证修改挤出机参数时，未勾选覆盖的耗材自动继承新值

**步骤**:
1. 取消所有耗材的覆盖
2. 修改 E1 的 retraction_length 从 0.8 改为 1.6
3. 切片
4. 验证

**预期结果**:
- F1 的 retraction_length = 1.6（直接使用 E1）
- F5 的 retraction_length = 1.6（继承 E1）
- F2-F4, F6-F8 保持原来的挤出机值

**通过**: [ ] 是 [ ] 否

---

### 测试 4: 覆盖不受挤出机修改影响

**目的**: 验证勾选覆盖的参数不受挤出机修改影响

**步骤**:
1. 为 F5 启用 z_hop 覆盖，   设置为 0.8
2. 修改 E1 的 z_hop 从 0.2 改为 0.6
3. 切片
4. 验证

**预期结果**:
- F5 的 z_hop = 0.8（使用覆盖值，**不**受 E1 修改影响）
- F1 的 z_hop = 0.6（使用 E1 新值）
- F6 的 z_hop = 0.3（继承 E2，未修改）

**通过**: [ ] 是 [ ] 否

---

### 测试 5: 参数独立性

**目的**: 验证修改一个参数不影响其他参数

**步骤**:
1. 取消所有耗材的覆盖
2. 为 F5 启用 wipe_distance 覆盖，
   设置为 50
3. 验证 z_hop_types 未受影响

**预期结果**:
- F5 的 wipe_distance = 50（使用覆盖值）
- F5 的 z_hop_types = Normal（仍继承 E1，**不**被清除）

**通过**: [ ] 是 [ ] 否

---

### 测试 6: 全部勾选覆盖

**目的**: 验证所有耗材都勾选覆盖时正常工作

**步骤**:
1. 为所有耗材 F1-F8 都启用 retraction_length 覆盖
2. 设置不同的值: F1=0.5, F2=0.6, F3=0.7, F4=0.8, F5=0.9, F6=1.0, F7=1.1, F8=1.2
3. 修改 E1 的 retraction_length 为 5.0
4. 切片
5. 验证

**预期结果**:
- 所有 F1-F8 都使用自己的覆盖值
- 没有任何耗材受 E1 修改影响

**通过**: [ ] 是 [ ] 否

## 验证方法

### 使用 analyze_gcode_params.py

```bash
python scripts/analyze_gcode_params.py output.gcode
```

### 检查日志

在切片日志中搜索:
- `DEBUG_MAP_INDICES_RESULT` - 验证映射索引
- `DEBUG_USER_OVERRIDE_FOUND` - 验证覆盖检测
- `DEBUG_FILAMENT_OVERRIDE_KEEP` - 验证参数保留
- `DEBUG_FILAMENT_OVERRIDE_ERASE` - 验证参数清除
